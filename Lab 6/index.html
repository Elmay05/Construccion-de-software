¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Para garantizar que los datos que se envian coinciden con los requisitos establecidos en los controels de formulario, de igual manera puedes realizar correcciones guiadas como en el caso del correo electronico, que siga el formato con @

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Esto se puede hacer con algunas palabras reservadas de html como entrar a la consola y en el formulario poner novalidate="" o el el submit poner formnovalidate=""

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Pues es buena practica dar una retroalimentacion al usuario en elmomento sobre los errores, o dar una guia de lo que tiene que hacer siendo este mas un complemento que una validacion de formulario como en este caso, el usuario, para garantizar una mejor seguridad y la proteccion de los datos, es bueno combinar esto con un servidor
